Підсумок для студентів:  
Ваше завдання:

1. Використати архітектуру, представлену на діаграмі, і задеплоїти всі сервіси: бекенд, PostgreSQL, Redis і фронтенд через S3 та CloudFront.
2. Забезпечити обробку 8 запитів за секунду у звичайних умовах та можливість масштабування під час пікових періодів.
3. Провести аналіз витрат для того, щоб вписатися в бюджет $100 на місяць для середовища розробки та продакшн.
4. Налаштувати CI/CD пайплайни для автоматизації процесів деплою для бекенду і фронтенду.
5. Впровадити надійні заходи безпеки для захисту даних користувачів і транзакцій.

Важливо: Не змінюй код. Код написано правильно, і твоя задача його задеплоїти, щоб виконати задачу цього проекту. Тому тобі не треба дописувати чи переписувати код, він вже робочий 

Лайфхак:  
Для вибору оптимального інстансу EC2 для обробки 8 запитів за секунду, скористайтеся ChatGPT. Він допоможе вам проаналізувати навантаження та підібрати відповідний тип EC2 інстансу, наприклад, t3.micro, t3.small або t3.medium, залежно від потреб вашого додатку і його ресурсомісткості.

Лайфхак 2:  
Використовуй ChatGPT скрізь :)  
Але не проси його зробити код за тебе чи рішення. Максимально використовуй його спочатку, щоб зрозуміти діаграму, зрозуміти всі взаємозв’язки на ній та чому саме так все зроблено.  
Далі використовуй його, коли будеш проходити курси, шукаючи ту інформацію, що тобі треба для реалізації проекту. Тут використовуй його, щоб глибше розуміти, більше створюй прикладів та аналогій.

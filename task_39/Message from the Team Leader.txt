Повідомлення від Тім Ліда (Олег Степаненко)  
Дата: 24 вересня 2024 року

Огляд проекту:  
Ми розробляємо платформу для бронювання вілл для відпочинку, де користувачі можуть переглядати доступні вілли, бронювати їх для відпочинку або святкувань та отримувати підтвердження. Додаток повинен мати надійний бекенд для управління користувачами, віллами та бронюваннями, а також статичний фронтенд для відображення наявних вілл і їх доступності. API-сервіс буде обробляти всі запити від користувачів і взаємодіяти з базою даних та системами кешування.

Задача:  
Ваше завдання — задеплоїти додаток, використовуючи надані сервіси згідно з архітектурою (зображеною на діаграмі). Бекенд, PostgreSQL та Redis повинні бути докеризовані та запущені через Docker-Compose на EC2. Статичний контент (фронтенд) має зберігатися на S3 і бути закритим через CloudFront для швидшого завантаження.

База даних:  
Для управління інформацією про користувачів, вілли та бронювання використовується PostgreSQL. Redis буде використовуватися для кешування сесій користувачів і прискорення пошукових запитів.

Докеризація:  
Усі сервіси повинні бути докеризовані для легкого масштабування та зручності управління. Docker-Compose використовується для запуску контейнерів з PostgreSQL, Redis і бекендом.

CI/CD процес:  
Впровадьте окремі CI/CD пайплайни для автоматизації деплою бекенду та фронтенду. Рішення повинно бути зручним для масштабування.

Важливі нотатки:  
- Статичний контент зберігається на S3 і кешується через CloudFront для прискорення завантаження сторінок.
- Бекенд, PostgreSQL і Redis запускаються через Docker-Compose на EC2, як показано на діаграмі.
- Потрібно забезпечити економічну ефективність рішення, щоб залишитися в межах бюджету.
